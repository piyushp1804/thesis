\section{Experimental Setup and Validation Methodology}
\label{sec:results:validation}

This section records the exact hardware, software, seeds, and
statistical protocol used to produce every number in the rest of
the chapter. The aim is that a reader with access to the
repository tag \texttt{phase-8b-complete} can reproduce any cell
of every table exactly.

\subsection{Hardware and Software Environment}
\label{sec:results:setup:env}

All runs reported here were executed on a 2024 Apple MacBook~Air
with an M3 processor (8 performance cores), 16~GiB unified RAM,
macOS 14.6.1, and a local Python 3.14.1 virtual environment. No
run made use of a GPU: the neural-surrogate MLP is small enough
to train in under two minutes on the M3 CPU, and the PPO agent
fits comfortably in the same regime. Long-running batch sweeps
for the 25-bar and 72-bar benchmarks were executed in parallel on
a Linux sandbox (Ubuntu 22.04, x86\_64, 8~GiB RAM) via the same
pinned dependency set; results from the two machines match to
within the floating-point tolerance of the tests. Exact pinned
versions of every dependency are recorded in
\texttt{requirements.txt} at the tag above; the headline
versions are numpy 2.4.4, scipy 1.17.1, pymoo 0.6.1.6, PyTorch
2.11.0, Stable-Baselines3 2.8.0, Gymnasium 1.2.3, and the
Anthropic SDK 0.96.0.

\subsection{Reproducibility Protocol}
\label{sec:results:setup:repro}

Every script accepts an explicit \texttt{--seed} argument that is
propagated into numpy, pymoo, and PyTorch; no global
\texttt{np.random.seed} is ever set. Each benchmark/algorithm run
is written to an \texttt{OptimizationResult} pickle keyed by
\texttt{\{benchmark\}\_\{algorithm\}\_seed\{SEED\}.pkl}, and the
results folder further contains a \texttt{scoreboard.csv} that
aggregates best, mean, and standard deviation per
(benchmark~$\times$~algorithm) across all available seeds. LLM
API responses are hashed by prompt content and cached under
\texttt{results/llm\_cache/} so that any reader can replay a
warm-started run offline without re-billing the Anthropic API.

Figure~\ref{fig:results:pipeline} sketches the end-to-end data
flow: benchmark definitions drive a pymoo-wrapped optimiser,
which calls either the finite-element solver or the neural
surrogate on each candidate, with every reported optimum routed
through the IS~800:2007 compliance module before being written to
\texttt{results/}.

\subsection{Evaluation Metrics}
\label{sec:results:setup:metrics}

For each (benchmark, algorithm) cell we report:
\begin{itemize}
  \item \textbf{Best weight}: the minimum feasible
        $W(\mathbf{x}^\star)$ observed across all seeds in the cell.
  \item \textbf{Mean $\pm$ std weight}: arithmetic mean and
        sample standard deviation of the per-seed best feasible
        weights.
  \item \textbf{Convergence generation}: the first generation at
        which the best feasible weight comes within $\num{1}\,\%$
        of the per-run final value (used only for comparisons of
        optimiser progress, e.g.\ the LLM warm-start study of
        \S\ref{sec:results:llm}).
  \item \textbf{Wall-clock per evaluation}: mean seconds per
        \texttt{evaluate(x)} call of the chosen evaluator,
        measured by an ensemble of 1000 random designs.
  \item \textbf{Feasibility rate}: fraction of seed runs whose
        reported optimum satisfies every IS~800 clause at
        strict tolerance.
\end{itemize}

\subsection{Statistical Protocol}
\label{sec:results:setup:stats}

Cross-seed variability is reported as mean $\pm$ standard
deviation across the available seeds per cell. For A/B comparisons
between paired runs on the same seed (LLM warm-start vs.\ random
initialisation, surrogate vs.\ FEM on identical designs), we use
the two-sided paired Wilcoxon signed-rank test and report the
$p$-value alongside the effect size. With $n$ seeds the test is
pragmatic rather than conservative --- for $n=\num{3}$ it
achieves $p$ values no smaller than $0.25$ --- so the seed counts
reported in each table should be read as part of the evidence
strength. Where $n<5$, we flag the result accordingly in the
surrounding prose; where $n\ge 10$, paired Wilcoxon reaches
conventional significance on effect sizes above approximately
$\num{15}~\%$.

\subsection{Acceptance Tolerances}
\label{sec:results:setup:tol}

The Phase~1 validation gate set the following per-benchmark
tolerances relative to the published literature optimum: 10-bar
\num{2.0}\,\%, 25-bar \num{2.5}\,\%, 72-bar \num{2.5}\,\%,
200-bar \num{3.0}\,\%. These are the sizes at which the original
source papers' optima are themselves inconsistent in the third
significant figure across independent re-runs in the published
literature, and therefore the size below which ``matching the
literature'' ceases to be a meaningful claim. The 10-bar, 25-bar,
and 72-bar results that follow clear their respective tolerances
handsomely (\num{0.004}\,\%, \num{0.008}\,\%, and
------~with the caveat that the 72-bar ``tolerance'' must be
interpreted against the \emph{rigorously-feasible} optimum of
\num{549}~lb rather than the literature-cited \num{379.62}~lb;
see \cref{sec:results:72bar} for the full discussion). The
200-bar benchmark is scaffolded but not reported in this thesis
and is listed as future work in \cref{sec:conclusion:future}.

\section{72-Bar Spatial Tower}
\label{sec:results:72bar}

The 72-bar spatial tower of Fleury and Schmit~\cite{fleury1980},
popularised in Erbatur et al.~\cite{erbatur2000} and Camp \&
Bichon~\cite{camp2004}, is a four-storey cantilever tower with $120
\times 120$~in square plan and 240~in total height. 20 nodes, 72 bars,
16 symmetry groups arranged per storey as legs (4), face diagonals
(8), top horizontals (4), plan diagonals (2) — the canonical Camp
ordering, so published designs from the literature can be plugged in
directly. The canonical benchmark carries two load cases: an
asymmetric tip load (LC1) and a uniform vertical compression at all
four tip nodes (LC2). The displacement constraint applies only to the
lateral ($x$ and $y$) components at the four tip nodes (12--15);
vertical and lower-storey displacements are unconstrained. Parameters
are listed in \cref{tab:results:72bar-setup}.

All three algorithms converge to the same neighbourhood of $\sim
\num{550}$~lb with cross-seed spreads under $\num{1}\,\%$, well above
the literature-cited minimum of $\num{379.62}$~lb. This gap warrants
explanation. We adopt the canonical Camp \& Bichon~\cite{camp2004}
problem definition exactly: identical group order, lateral-tip-only
displacement check, and the two standard load cases. Yet plugging the
published 380.24~lb ant-colony optimum from~\cite{camp2004} directly
into our finite-element model yields a maximum lateral tip
displacement of \num{0.41}~in — \num{62}\,\% above the
$\num{0.25}$~in limit — and a maximum member stress of \num{30503}~psi
— \num{22}\,\% above the $\num{25000}$~psi limit. The same is true of
the Bekdaş~\cite{bekdas2015} continuous optimum at \num{379.6}~lb.
The published literature minima are therefore \emph{infeasible} under
rigorous constraint enforcement; they are reachable only with soft
penalty handling that admits violations of the order of \num{20}--%
\num{25}\,\%. With a hard ($g \le 0$) constraint, the true feasible
optimum sits near $\num{550}$~lb. Particle-swarm optimisation reaches
this floor with effectively zero cross-seed variance; GA and NSGA-II
agree to within $\num{1}\,\%$. The percent-error column in
\cref{tab:results:72bar} should therefore be read as the gap between
the rigorous-feasibility optimum and the soft-penalty literature
value, not as an optimiser shortfall — a methodologically important
distinction that, to our knowledge, has not been called out in the
prior 72-bar literature.

\section{200-Bar Planar Truss}
\label{sec:results:200bar}

The 200-bar planar steel truss was scaled-matched to the benchmark
of Kaveh and Talatahari~\cite{kaveh2010} but encoded here as a
principled regular-grid stepped tower in
\texttt{src/benchmarks/truss\_200bar.py}: 77 nodes, 200 bars, 29
design groups, 30~m total height, three load cases (lateral wind
top, asymmetric mid-span, vertical at tip). This is the only
benchmark in the thesis whose problem definition is rigorously
in the \emph{Indian civil-engineering} setting --- SI units,
structural steel ($E = \num{210}$~GPa, $\rho = \num{7850}$~kg/m³),
250~MPa stress envelope --- so it also end-to-end exercises the
IS~800:2007 compliance layer (\cref{sec:results:is800}) against a
real-material problem, not the imperial/aluminium idealisation
of the earlier three benchmarks. The module sets
\texttt{reference\_verified = False} and we do not claim a
literature match against the Kaveh 25{,}445~kg number: our
regular-grid node layout differs from the exact Kaveh
coordinates, and we report only our own rigorous-FEM optimum.
Parameters are listed in \cref{tab:results:200bar-setup}.

statistically indistinguishable at the \num{5}\,\% level by a
two-sample Wilcoxon test ($p = 0.69$). At every reported optimum
the maximum member stress sits within \num{0.05}\,\% of the
250~MPa envelope and the maximum nodal displacement is
\num{0.033}~m --- only \num{33}\,\% of the \num{0.10}~m
deflection limit --- so the 200-bar optimum is cleanly stress-
binding rather than deflection-binding, exactly the regime in
which the IS~800:2007 tension/compression checks
(\cref{sec:results:is800}) become decisive.

NSGA-II is reported for completeness but should not be read as a
competing single-objective number. Its task is to trace a Pareto
front between weight and maximum displacement, and the
weight-minimum extreme of that front lies at designs that still
satisfy both constraints but have been pulled off the stress
boundary by the displacement objective. The three NSGA-II Pareto
fronts each contain 100 non-dominated points, giving the designer
a genuine weight/deflection tradeoff curve if vertical stiffness
matters more than raw mass; its role is complementary to, not
competitive with, GA and PSO.

Per-seed wall time is \num{2.8}--\num{6.7} minutes on an Apple
M3 (CPU only, pure-Python FEM), with the first GA seed carrying a
\num{15}-minute one-off JIT warm-up cost that falls to under
four minutes per seed for the remainder. The total compute
budget for
\cref{tab:results:200bar} is \num{57} wall-clock minutes for the
full thirteen-seed matrix. The 29-variable 200-bar benchmark thus
completes in roughly the same wall time per seed as 72-bar even
though its design vector has nearly double the dimensionality and
its element count is \num{2.8}$\times$ larger, confirming that the
framework scales past 72-bar without code changes, without
surrogate assistance, and without any algorithm-specific tuning.

\section{Neural Surrogate Accuracy and Speedup}
\label{sec:results:surrogate}

The surrogate layer of \cref{sec:method:surrogate} was trained on
the 10-bar benchmark using a Latin-hypercube sample of
\num{10000} designs, of which \num{1000} were held out as a test
set. Training converged in \num{117}~epochs at an Adam learning
rate of $\num{e-3}$ under a cosine schedule, taking
\num{112}~seconds on the M3 CPU. The trained checkpoint is
\texttt{results/10bar\_surrogate.pt} at \num{144}~KB on disk.

\subsection{Held-Out Accuracy}
\label{sec:results:surrogate:accuracy}

\Cref{tab:results:surrogate-r2} reports the coefficient of
determination $R^2$ per output head on the held-out test set.
The weight head achieves $R^2=\num{0.9993}$ --- essentially
identity-function quality --- while the stress and displacement
heads reach $\num{0.81}$ and $\num{0.87}$ respectively. This
asymmetry is a direct consequence of the benchmark's constraint
landscape: weight is a smooth linear function of the design
vector and is easy to fit, whereas the maximum stress and
displacement over ten members are discontinuous in areas as the
member carrying the max switches, and the MLP cannot track those
discontinuities perfectly from finite training data. The pattern
matches the stress/displacement-harder-than-weight observation
reported by Persia et al.~\cite{persia2025} and Sun et
al.~\cite{sun2023} on comparable surrogate problems.

\subsection{Speedup in the Optimiser Inner Loop}
\label{sec:results:surrogate:speedup}

Used inside the pymoo inner loop as a drop-in \texttt{evaluate(x)}
replacement, the surrogate evaluates a \num{100}-member
population in \num{0.031}~s batched, versus the FEM engine's
\num{4.1}~s per population (\num{41}~ms/design, unbatched). This
is a $\num{132}\times$ wall-clock speedup, comfortably exceeding
the $\num{50}\times$ target set in \S\ref{sec:intro:objectives}
(objective~3). Running GA + surrogate for the full
\num{500}-generation budget completes in \num{17}~s end-to-end;
the same run on FEM takes \num{34}~min. The surrogate-in-loop
optima match the FEM-in-loop optima to within
$\num{0.15}\,\%$ on weight across ten independent seeds,
confirming that the weight-$R^2$ accuracy is enough to preserve
the optimiser's trajectory even though the stress and
displacement heads are less accurate.

\subsection{Use as Feasibility Screen}
\label{sec:results:surrogate:feasibility}

Because the stress and displacement heads are less accurate, we
deploy the surrogate in a hybrid adaptive-sampling pattern in
the spirit of Queipo et al.~\cite{queipo2005}: the surrogate is
used unconditionally for weight, and only as a pre-screen for
feasibility. Designs flagged as clearly infeasible by the
surrogate ($\hat{\sigma}_{\max} > \num{1.3}\,\sigma_\text{lim}$
or $\hat{\delta}_{\max} > \num{1.3}\,\delta_\text{lim}$) are
rejected without a FEM call; designs that pass this screen are
re-evaluated with FEM to confirm actual feasibility. In
practice, the screen correctly rejects $\approx\num{82}\,\%$ of
infeasible candidates during the early generations without any
false negatives relative to the FEM ground truth.

\section{PPO Agent Performance}
\label{sec:results:rl}

The reinforcement-learning agent of \cref{sec:method:rl} was
trained on the 10-bar benchmark against the surrogate evaluator
for a rollout budget of \num{200000} environment steps
(\num{18}~minutes on the M3 CPU). Evaluation combines one
deterministic rollout with \num{19} stochastic rollouts of the
trained policy; each proposed design is re-evaluated with the
exact FEM engine before being scored for feasibility.

\subsection{Achieved Weight}
\label{sec:results:rl:weight}

The best design proposed by the agent weighs \num{5876.9}~lb,
against the \num{5060.85}~lb literature optimum --- a
$\num{16.12}\,\%$ gap. This misses the $\num{5}\,\%$ validation
gate stated in \S\ref{sec:intro:objectives} (objective~4), and
the miss is the honest result we report rather than a headline
contribution. It is worth stating plainly what this result does
and does not mean. It does not mean the PPO agent cannot reach
the 10-bar optimum in principle; with a larger rollout budget
(approximately \num{e6} steps) and careful reward shaping the
literature~\cite{hayashi2020rl} has reported closer agreement.
It does mean that, on this single benchmark instance at the
compute budget available, a vanilla Stable-Baselines3 PPO agent
does not beat the tuned GA and PSO baselines of
\cref{sec:results:10bar}.

The observation is consistent with Zhao et
al.~\cite{zhao2021rl}'s broader thesis about model-free RL on
single-instance structural optimisation: the method's advantage
lies in the transferability of the learned policy across
problem instances rather than in matching a tuned classical
optimiser on one instance. We return to this framing in the
Discussion (\cref{sec:results:discussion}).

\subsection{Inference-Time Advantage}
\label{sec:results:rl:inference}

The genuinely useful number for this layer is the
\emph{inference time}: the trained policy proposes a design in a
single forward pass of roughly \num{0.2}~s, versus the
$\sim$\num{17}--\num{30}~s of a fresh surrogate-in-loop GA run and
the $\sim$\num{34}~min of a fresh FEM-in-loop GA run.
\Cref{tab:results:rl-inference} makes this explicit. For
interactive design exploration --- the Streamlit UI of
\cref{sec:method:ui} --- the policy provides a tunable-at-
millisecond-latency first-cut design that the user can then
refine through a full optimisation run if desired.

\section{LLM Warm-Start Study}
\label{sec:results:llm}

We evaluate a large-language-model (LLM) as a semantic initialiser for
the genetic algorithm. Rather than replacing any component of the
optimisation loop, the LLM is asked — via the Anthropic Claude API~%
\cite{anthropic-docs} — to propose a single initial design vector from
a natural-language description of the benchmark geometry, loads, and
constraints; this vector seeds the first \num{10}\,\% of the GA
population while the remaining \num{90}\,\% is drawn uniformly at
random. Every API response is hashed and cached under
\texttt{results/llm\_cache/}, making the comparison fully reproducible
without re-billing the API across seeds.

On the 10-bar benchmark, Claude Sonnet~4.5 correctly identified the
redundant-member set \{2, 5, 6, 10\} on its first call and proposed
areas within \num{10}\,\% of Sunar's published optimum with confidence
\num{0.82}. Paired Wilcoxon tests over 40 independent seeds showed
a \num{36.8}\,\% reduction in generations-to-90-percent-convergence
relative to random initialisation, at $p = 0.0046$.

On 25-bar, Claude's first-call proposal achieves a 76.6\% reduction

random init sometimes settles into. With only three seeds the

would very likely pull the $p$-value under 0.05 given the effect size.

\cref{sec:results:72bar}), the convergence target is anchored
adaptively to $\num{1.02} \times \max(W_\text{final,random})
= \num{564.5}$~lb because the literature-cited \num{379.62}~lb target
is unreachable under rigorous constraint enforcement. Both arms
converge briskly within the 200-generation budget — random in
\num{46.2} generations on average, LLM-warmstarted in \num{48.4} —
and the LLM-warmstart arm is in fact $\num{4.8}\,\%$ \emph{slower},
with $p = 0.81$ (Wilcoxon, $n=5$): a flat null result. The LLM's
suggested design (tapered from a base-heavy
$\sim 1.8$~in\textsuperscript{2} down to
$\sim 0.2$~in\textsuperscript{2} at the tip) is physically plausible
and is in the right basin of attraction, but on a 16-variable problem
with two simultaneous load cases the GA's local search finds the
constraint boundary in 40--60 generations from either start. The
qualitative architectural insight that helped on 10-bar (eliminating
the redundant members \{2,5,6,10\}) has no analogue on the
fully-occupied 72-bar tower.

This null result is itself the headline finding for the LLM-warmstart
study: the technique helps the most on problems where the LLM can
identify a structural insight that the GA would otherwise have to
discover through trial-and-error (10-bar redundant members,
\num{36.8}\,\% reduction, $p < 0.005$), and progressively less on
problems where the search space is small enough that the GA finds
the binding constraint quickly without help.

Each cached Claude response is under 1~KB and is committed to the
repository so that any reader can reproduce the warm-started runs by
setting \texttt{use\_cache=True} without an API key of their own.

\section{Multi-Objective Pareto Fronts}
\label{sec:results:pareto}

The single-objective weight-minimisation results reported in
\S\S\ref{sec:results:10bar}--\ref{sec:results:72bar} are a narrow
slice of what an engineer working through a real design
iteration actually needs. Once the weight is competitive, the
next question is usually \emph{how sensitive is that weight to
the serviceability deflection requirement?} --- reducing the
allowed deflection stiffens the structure and increases its
weight, and a designer working against a brittle-cladding
limit $L/\num{325}$ would like to know the shape of that
trade-off curve before committing to a target.

NSGA-II~\cite{deb2002nsga2} produces exactly that curve in one
run. We reformulate the \texttt{TrussProblem} adapter with two
objectives --- minimise weight, minimise maximum nodal
displacement --- and drop the displacement constraint (retaining
only the stress and IS~800 checks), running NSGA-II with
population \num{100} for \num{300} generations across three
seeds on each benchmark.

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.80\textwidth]{fig_4_2_2_10bar_pareto.png}
  \caption{10-bar weight vs maximum-displacement Pareto front
    from NSGA-II. Three seeds are overlaid; each curve is the
    non-dominated set returned at generation \num{300}. The
    ``knee'' region around $W\approx\num{5100}$~lb and
    $\delta_{\max}\approx\num{2.0}$~in corresponds to the
    single-objective optimum of
    \cref{tab:results:10bar}.}
  \label{fig:results:10bar-pareto}
\end{figure}

Three qualitative observations from the Pareto fronts generalise
across the three benchmarks:
\begin{itemize}
  \item The front is smooth and continuous on 10-bar and 25-bar,
        reflecting that weight and displacement are both
        well-behaved functions of the design vector once the
        stress constraints are satisfied;
  \item The front exhibits a pronounced knee roughly at the
        position of the single-objective optimum, so that
        loosening the displacement limit beyond that point buys
        very little weight reduction and tightening it below
        that point costs weight sharply. For 10-bar, the knee
        sits at $W\approx\num{5100}$~lb and
        $\delta_{\max}\approx\num{2.0}$~in;
  \item At the cheap-weight end of the front the surrogate's
        displacement $R^2\approx\num{0.87}$ is insufficient to
        rank points to the accuracy needed for a Pareto
        comparison, so all Pareto runs were performed against
        the exact FEM engine rather than the surrogate.
\end{itemize}

Each front contains between \num{24} and \num{32} non-dominated
points, comfortably clearing the \num{20}-point gate of
\S\ref{sec:intro:objectives} (objective~2).

\section{Ablation Studies}
\label{sec:results:ablation}

Three ablations isolate the contribution of individual layers of
the stack to the overall numerical outcome.

\subsection{FEM vs Surrogate in the Inner Loop}
\label{sec:results:ablation:surrogate}

The first ablation replaces the FEM evaluator inside the pymoo
inner loop with the trained MLP surrogate of
\cref{sec:method:surrogate}, holding every other setting fixed.
On 10-bar across ten seeds, the mean best weight changes from
\num{5075.92}~lb (FEM, cross-seed std $\num{7.11}$~lb) to
\num{5080.34}~lb (surrogate, cross-seed std $\num{8.92}$~lb),
a $\num{0.09}\,\%$ shift that is smaller than the cross-seed
std of either arm. Wall-clock per full \num{500}-generation run
drops from $\num{34}$~min to $\num{17}$~s, a
$\num{120}\times$ speedup. The trade-off is unambiguously worth
making when weight is the target; the parity plot of
\cref{fig:results:surrogate-parity} visualises why.

\subsection{Hard vs Soft Constraint Handling}
\label{sec:results:ablation:constraints}

The second ablation --- and the most important numerical finding
of the thesis --- concerns the constraint handling on 72-bar.
Two versions of the optimisation were run: one with strict hard
feasibility ($g_i(\mathbf{x}) \le 0$ for every IS~800 clause, a
design is feasible if and only if all $g_i \le 0$), and one with
a soft quadratic penalty where $g_i>0$ is accepted but penalised
at $\lambda = \num{e4}\,g_i^2$. The soft-penalty formulation is
what Camp \&~Bichon~\cite{camp2004} and Bekdas et
al.~\cite{bekdas2015} report, and under it our GA reaches
$\num{379}$--$\num{382}$~lb (matching their published values to
within \num{0.5}\,\%). Under strict $g \le 0$, the GA reaches
$\num{549.78}$~lb, because any design under $\num{400}$~lb
violates the lateral-displacement constraint at the tip nodes by
$\num{22}$--$\num{62}\,\%$. \Cref{fig:results:hard-vs-soft}
overlays the two convergence trajectories.

\begin{figure}[htbp]
  \centering
  \includegraphics[width=0.95\textwidth]{fig_4_5_2_hard_vs_soft.png}
  \caption{72-bar convergence with hard vs soft constraint
    handling. Solid blue: strict $g \leq 0$ (this work);
    dashed red: large-penalty soft handling (as reported by
    Camp \&~Bichon, 2004). The shaded region marks the
    $\sim\num{25}\,\%$ residual infeasibility range of the
    literature-cited optimum under rigorous re-evaluation.}
  \label{fig:results:hard-vs-soft}
\end{figure}

The $\num{170}$-lb gap between \num{380}~lb and \num{550}~lb is
therefore an artefact of the constraint-handling convention, not
an optimiser shortfall. The thesis's contribution here is to
make that gap visible and to recommend that future 72-bar
literature separate the two quantities rather than reporting
them under a single ``optimum weight'' heading.

\subsection{IS~800:2007 Inclusion vs Exclusion}
\label{sec:results:ablation:is800}

The third ablation drops the IS~800 checks entirely, leaving
only the original stress and displacement constraints from the
benchmark definition. On 10-bar this changes the best weight by
less than $\num{0.2}\,\%$, because the original Sunar stress
limit of $\pm\num{25000}$~psi already binds below the IS~800
compression-buckling capacity of every member in the optimum.
On 25-bar the best weight decreases by $\num{0.3}\,\%$, and on
72-bar by a more substantial $\num{2.7}\,\%$: the IS~800 Cl.~7.1
buckling check \emph{is} binding on several tower legs in the
rigorous-feasibility optimum, so including it raises the
achievable weight by approximately $\num{15}$~lb. The practical
takeaway is that IS~800 compliance is a cheap addition on
small-scale benchmarks but becomes a first-order design driver
as the structure scales, consistent with the design-office
intuition that it is the secondary-buckling checks that govern
real transmission towers.

\Cref{fig:results:pareto-is800} illustrates the shift of the
Pareto front under the same ablation on 10-bar; the two fronts
are nearly parallel, offset by a near-constant $\num{90}$-lb
weight penalty attributable to the IS~800 buckling constraint.

\subsection{GA Population Size}
\label{sec:results:sensitivity:pop}

hyperparameter. \Cref{fig:results:pop-size} overlays the 10-bar

recommendation for future work is pop~\num{100} for 10-bar and

\section{Discussion}
\label{sec:results:discussion}

The numerical results in the preceding sections support four
headline claims that we collect here.

\paragraph{The classical pipeline is correct.} The match between
our PSO-best on 10-bar (\num{5061.05}~lb) and Sunar \&
Belegundu's 1991 optimum (\num{5060.85}~lb) to four decimal
places is the single strongest sanity-check in the thesis. It
cannot occur by coincidence: reproducing a thirty-year-old
published optimum to that tolerance requires that the FEM is
correct, the benchmark encoding is correct, the constraint
handling is correct, and the optimiser wiring is correct. The
25-bar match at $\num{0.008}\,\%$ confirms the same for the 3D
spatial case. Every subsequent layer of the stack ultimately
composes with this classical core; getting it right is the
foundation that lets the rest of the work stand.

\paragraph{The 72-bar discrepancy is an encoding-convention
finding, not an optimiser shortfall.} The $\num{44}$--\num{46}\,\%
gap between our rigorously-feasible 72-bar optimum of
$\num{549}$~lb and the literature-cited \num{379.62}~lb is
uniformly reproducible across GA, PSO, and NSGA-II and persists
across all seeds. Once the published \num{379.62}~lb optimum is
plugged into our FEM engine and fails every one of the
lateral-displacement and stress constraints by \num{22}--%
\num{62}\,\%, the source of the gap becomes apparent: the
literature's soft-penalty formulation admits constraint
violations at a level that our strict $g\le 0$ formulation does
not. We do not argue that the literature is wrong; rather that
72-bar optima should be reported with an explicit statement of
the constraint tolerance used, and that the difference between
\num{380}~lb and \num{550}~lb is economically and
structurally significant enough to deserve that clarity. The
ablation of \cref{sec:results:ablation:constraints} makes the
point numerically; \cref{fig:results:hard-vs-soft} makes it
visually.

\paragraph{LLM warm-start helps where the LLM can see a
structural insight.} The \num{36.8}\,\% reduction in
generations-to-convergence on 10-bar (\num{40} seeds,
$p = \num{0.0046}$) is driven by Claude correctly identifying
the four redundant members (2, 5, 6, 10) and proposing an
initial design with their areas at the lower bound. On 25-bar
the model similarly recognises the symmetry groups and proposes
a plausible tapered tower section; on 72-bar, however, no
comparable insight is available --- the tower has no redundant
members in the strict sense, and a GA finds the binding
constraint boundary in \num{50}-ish generations whether warm-
started or not. The headline is not that LLMs help uniformly but
that they help where a human expert would recognise a
qualitative architectural insight. This matches Makatura et
al.~\cite{makatura2024}'s framing of the LLM as a
\emph{co-designer} that offers structural suggestions rather
than as a replacement for the optimiser.

\paragraph{The surrogate is the biggest single speedup in the
stack.} At $\num{100}$--$\num{150}\times$ wall-clock speedup with
weight-$R^2=\num{0.9993}$ and a solution that matches the FEM-in-
loop optimum to $\num{0.09}\,\%$, the neural surrogate
unambiguously pays for itself even on a single run: the
one-time training cost of $\num{112}$~s amortises into a single
surrogate-in-loop optimisation that would otherwise take
\num{34}~min. The reinforcement-learning agent at +\num{16}\,\%
over the literature optimum does not beat tuned classical
solvers on single instances, but its \num{0.2}-s inference
latency fills a different niche --- real-time interactive
exploration in the Streamlit UI. Taken together, the AI layers
address two orthogonal bottlenecks in applied structural
optimisation: the inner-loop evaluation cost (surrogate) and the
cold-start latency of a fresh optimisation run (RL and LLM
warm-start).

\paragraph{Practical implications for Indian design offices.}
The thesis is framed around Indian infrastructure, and the
IS~800:2007 compliance layer is the concrete bridge from
benchmark optimisation to design-office relevance. The
$\num{34}$-min-to-\num{17}-s reduction on 10-bar, combined with
a Streamlit UI that a practitioner can drive without writing
Python, is the smallest useful unit of deployment: a utility
that an engineer can run during a design meeting and obtain a
feasible optimum, plus a confidence-ranked LLM-suggested design
to cross-check against intuition, before the meeting ends. The
end-to-end demo on 10-bar meets this under-\num{10}-s target
(see \cref{sec:method:ui}).

\section{Threats to Validity}
\label{sec:results:threats}

We close the chapter by explicitly listing the threats to the
validity of the numerical claims above.

\paragraph{Linear elastic FEM.} The finite-element engine is
linear elastic and does not model geometric or material
non-linearity. For the pin-jointed small-displacement benchmarks
studied here this is well-justified, but a deployed design aid
would need post-buckling and large-deformation analyses before
being trusted on real transmission towers. Logan~\cite{logan2017fem}
and Cook et al.~\cite{cook2002fem} treat the non-linear
extensions, but they are out of scope for this thesis.

\paragraph{Small seed counts on 25-bar and 72-bar NSGA-II.} The
25-bar and 72-bar NSGA-II cells of the scoreboard use three
seeds, driven by the larger wall-clock per run of the
multi-objective formulation on 3D benchmarks. Three seeds are
sufficient to establish mean behaviour but not to detect tail
events; a future pass should extend to ten seeds minimum.

\paragraph{Single-instance RL.} The PPO agent was trained and
evaluated on one 10-bar instance. Zhao et
al.~\cite{zhao2021rl}'s transferability claim --- that the
value of RL is in cross-instance generalisation rather than
single-instance optimality --- has not been directly tested here
because all four benchmarks have different problem
dimensionalities and the agent's observation/action space is
tied to the 10-bar problem. A true transferability study would
require a shared observation encoder (a graph neural network
over the truss adjacency, following
Hayashi \& Ohsaki~\cite{hayashi2020rl}); this is listed in
\cref{sec:conclusion:future}.

\paragraph{IS~800 buckling-curve choice.} IS~800:2007 provides
four column-curve variants (a, b, c, d) corresponding to
different imperfection factors; we use curve~a throughout. The
choice is conservative for rolled I-sections and tubular
members, and matches the recommendation of
Subramanian~\cite{subramanian2008}, but a designer using the
framework on welded built-up sections should explicitly
re-parameterise for curve~c. This is a \num{10}-line code
change but it is not auto-detected.

\paragraph{Surrogate extrapolation at design-space corners.} The
MLP surrogate was trained on a Latin-hypercube sample that
covers the full bound box uniformly, but the achieved optima
often sit on the bound boundary (members 2, 5, 6, 10 on 10-bar
are at the $A=\num{0.1}$-in\textsuperscript{2} lower bound).
The stress and displacement $R^2$ values of $\num{0.81}$ and
$\num{0.87}$ are computed on the uniform test set; accuracy at
the bound-boundary corners is lower. This is the motivation for
the hybrid adaptive-sampling pattern of
\cref{sec:results:surrogate:feasibility} and for future work on
active-learning surrogate refinement
(\cref{sec:conclusion:future}).

\paragraph{LLM model drift.} The LLM warm-start results use
Claude Opus~4.7. As Anthropic releases successor models, the
cached designs in \texttt{results/llm\_cache/} will remain
reproducible but the numerical effect sizes may shift. The
thesis's methodological contribution --- the pipeline and the
experimental protocol --- is model-agnostic, but the specific
$\num{36.8}\,\%$ reduction should be re-measured on any
replacement model before being cited as a persistent number.
