## 1. What the thesis is about

M.Tech / IDD thesis (Aryan, IDD Civil Y5, IIT BHU;
supervisor Dr.~Krishna Kant Pathak): **LLM-warm-started
metaheuristics for structural (truss) optimization.**
Core claim: seeding GA / PSO / NSGA-II populations with designs generated by a
large language model (Claude Opus) converges faster than random initialization
on standard truss weight-minimization benchmarks under AISC / IS-800 stress &
displacement constraints.

Background docs in `docs/`:
- `thesis_vision.md` — what we're proving
- `thesis_explained_simply.md` — plain-English elevator pitch
- `ground_truth.md` — reference optima from the literature
- `cursor_playbook.md` — coding conventions

**Open issue on 72-bar:** our best is ~549 lb vs literature 379.62 lb. Gap is
consistent across all three solvers and all seeds → almost certainly an
**encoding / constraint mismatch**, not a solver weakness. Before Phase 8
writeup of 4.4, verify against the canonical Erbatur 2000 formulation:
- group membership (4 symmetry groups of 16/18/2/20 bars? check `truss_72bar.py`)
- loading (two load cases, not one)
- displacement limits (0.25 in on top nodes, all 6 DOF)
- discrete area set vs continuous

Chapter 4.4 currently presents these numbers honestly with a note flagging the
discrepancy; this is acceptable for a draft but should be resolved before submission.

LLM warm-start signal:
- 10-bar (40 seeds): gens-to-converge reduced 36.8% (significant, p=0.0046)
- 25-bar (3 seeds): large reduction on seed 42 (199→18 gens), small on others
- 72-bar (3 seeds): **no meaningful reduction** — both conditions hit 199-gen ceiling. Need more seeds and/or longer budget; likely related to the 44% optimality gap above.

## 6. How to run things

```bash
# Full test suite (24 min, 76 tests)
pytest -q

# Fast suite only (skip slow gates, ~2 min)
pytest -q -m "not slow"

# Recompile thesis PDF
cd thesis_writeup && latexmk -pdf -interaction=nonstopmode main.tex

# Re-run a benchmark batch
python scripts/run_batch.py --benchmark 25bar --algo ga --seeds 42 123 456 789 2024

# LLM warm-start (NEEDS ANTHROPIC_API_KEY in env)
export ANTHROPIC_API_KEY=sk-ant-...
python scripts/run_llm_warmstart.py --benchmark 25bar --seeds 42 123 456

# Refresh scoreboard.csv from *_summary.csv files
python scripts/aggregate_results.py
```

Cached LLM designs live in `results/llm_cache/` — delete a subdir to force re-query.
